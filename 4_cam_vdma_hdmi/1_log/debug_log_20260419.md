## 五、VDMA状态分析

### 5.1 VDMASR寄存器解析

**S2MM VDMASR = 0x00010000**:
- bit[12] = 1 → SOF中断状态（收到帧开始信号）
- bits[23:16] = 0x01 → 帧计数 = 1

**MM2S VDMASR = 0x00011000**:
- bits[23:16] = 0x01 → 帧计数 = 1

### 5.2 配置验证

| 寄存器 | 值 | 说明 |
|--------|-----|------|
| STRIDE | 0x780 (1920) | ✅ 正确 |
| HSIZE | 0x780 (1920) | ✅ 正确 |
| VSIZE | 0x1E0 (480) | ✅ 正确 |
| START_ADDR | 0x10000000 | ✅ 正确 |

## 八、cam_cap_data.v 关键逻辑

```verilog
// 帧计数器 - 需要10帧
localparam WAIT_FRAME = 10;

always@(posedge cam_pclk or negedge rst_n_sync)
    if(!rst_n_sync)
        frame_cnt <= 0;
    else if(pos_vsync)begin
        if(frame_cnt == WAIT_FRAME-1)
            frame_cnt <= frame_cnt;
        else 
            frame_cnt <= frame_cnt+1;
    end

// wait_done - 10帧后置1
always@(posedge cam_pclk or negedge rst_n_sync)
    if(!rst_n_sync)
        wait_done <= 0;
    else if(pos_vsync && frame_cnt == WAIT_FRAME-1)
        wait_done <= 1;

// RGB565 → RGB888转换
assign rgb888_data = {rgb565_data[15:11],3'b000,
                       rgb565_data[10:5],2'b00,
                       rgb565_data[4:0],3'b000};

// 输出 - wait_done=0时全为0
assign vid_data  = (wait_done) ? rgb888_data : 'd0;
assign vid_ce    = (wait_done) ? ((vid_active_video & data_flag_d0)||(~vid_active_video)) : 0;
assign vid_vsync = (wait_done) ? cam_vsync_d1 : 'd0;
assign vid_active_video = (wait_done) ? cam_href_d1 : 'd0;
```
